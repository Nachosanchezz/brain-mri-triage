Guía paso a paso — Repositorio para una CNN 3D de triaje cerebral (tumor / sano)

Enfoque: encoder 3D preentrenado (FOMO260K + pesos de AMAES) + fine-tuning con datos etiquetados. Documento de trabajo.

0. Idea del enfoque (en 4 líneas)

  1. Preentrenar = aprender "cómo es un cerebro en RM" a partir de muchísimos volúmenes sin etiqueta. No lo haces tú: descargas un encoder 3D ya preentrenado (AMAES, entrenado sobre FOMO260K: 260.927 scans de 910 fuentes).
  2. Fine-tuning = enseñarle TU tarea (tumor / sano) con un conjunto etiquetado.
  3. Para que aprenda la lesión y no "de qué dataset viene": que en los datos de fine-tuning cada clase venga de varias fuentes mezcladas (o ambas clases del mismo origen). Así "la fuente" deja de predecir la etiqueta y lo único consistente que separa las clases es el tumor.
  4. Resultado: una CNN 3D que distingue sano/tumor de verdad, y con N pequeño etiquetado basta porque el encoder ya sabe de cerebros.

3. El proceso, explicado claro (preentrenar vs fine-tuning)

FaseQué haceDe dónde sale
PreentrenamientoAprende rasgos generales de cerebro en RM con MUCHÍSIMOS volúmenes sin etiquetaNo lo haces tú: descargas los pesos de AMAES (entrenados sobre FOMO260K)
Fine-tuningAprende TU tarea concreta: tumor / sanoLo haces tú, con un set etiquetado
EvaluaciónMide si distingue de verdadConjunto de test separado
La condición clave: el preentrenamiento NO crea etiquetas; sigues necesitando un conjunto etiquetado para el fine-tuning. Lo que consigue es que ese conjunto, aunque sea pequeño, baste, porque el encoder ya sabe de cerebros.

4.2 Conseguir los datos etiquetados

Opción B (la preferente, si las etiquetas existen): FOMO260K.

  • Está en Hugging Face: FOMO-MRI/FOMO260K. Formato NIfTI (.nii.gz). Licencia CC-BY-NC-SA-4.0 (uso académico OK).
  • Trae metadatos: participants.tsv (diagnóstico: control / tumor…), mri_info.tsv, mapping.tsv.
  • Como mezcla 910 fuentes y ambas clases salen de fuentes solapadas, "la fuente" no predice la etiqueta → entrenar ahí fuerza a aprender la lesión.
# Login en Hugging Face (una sola vez)
huggingface-cli login

# NO descargues 1,87 TB enteros. Baja solo lo que necesites (subconjunto).
# Ejemplo: descargar los metadatos primero para ver las etiquetas
huggingface-cli download FOMO-MRI/FOMO260K --repo-type dataset \
  --include "*participants.tsv" --local-dir data/raw/fomo_meta
Verificar antes de seguir: abre los participants.tsv y comprueba que de verdad distinguen tumor vs control de forma usable. Si es así, descargas solo los sujetos de esas dos clases.

Opción A (alternativa segura): un set etiquetado pequeño de fuente única (o con varias fuentes por clase). Sirve para el fine-tuning aunque sea de decenas de sujetos, porque el encoder ya está preentrenado.

4.3 Preparar los datos (src/data/prepare.py)

  • Leer participants.tsv y construir data/processed/metadata.csv con columnas: ruta, label (1=tumor, 0=sano), fuente, subject_id.
  • Homogeneizar cada volumen: reorientar a RAS, remuestrear a 1 mm³, recortar/rellenar a una forma fija, normalizar (z-score sobre tejido). (El encoder de AMAES espera un preprocesado concreto: ajústate al que indique su repo.)

4.4 Repartir en train/val/test (src/data/splits.py)

  • Estratificado por clase, agrupado por subject_id (que ningún sujeto caiga en dos particiones), semilla fija (42), y congelar el split en un JSON.

4.5 Montar el modelo (src/models/)

  • encoder.py: carga el encoder 3D preentrenado de AMAES desde models/pretrained/.
  • classifier3d.py: encoder + una cabeza de clasificación (pooling global → capa lineal → 1 logit). Salida = sigmoid(logit) = probabilidad de tumor.

4.6 Fine-tuning (src/training/finetune.py)

  • Estrategia recomendada: empezar con el encoder congelado (entrenar solo la cabeza) y luego, si va bien, descongelar con learning rate bajo.
  • Pérdida BCEWithLogitsLoss; optimizador AdamW; guardar el mejor checkpoint por balanced accuracy con sensibilidad mínima; semilla fija; split congelado.

4.7 Evaluar (src/evaluation/evaluate.py)

  • AUC, PR-AUC, sensibilidad, especificidad, F1, matriz de confusión, e intervalos de confianza por bootstrap.
  • El umbral de decisión se elige en validación y se aplica a test.

5. Orden de ejecución (el flujo)

bash scripts/download_weights.sh           # 1) pesos de AMAES -> models/pretrained/
python -m src.data.prepare                 # 2) raw -> processed + metadata.csv
python -m src.data.splits                  # 3) train/val/test (congelado)
python -m src.training.finetune --config configs/finetune.yaml   # 4) fine-tuning
python -m src.evaluation.evaluate --checkpoint outputs/best.pt   # 5) métricas

6. Notas prácticas (para no estrellarte)

  • No preentrenes tú: descarga los pesos. Preentrenar 260k volúmenes es inviable en tu equipo; el fine-tuning sí es asumible (GPU AMD ROCm).
  • No bajes 1,87 TB: descarga metadatos primero y luego solo el subconjunto de sujetos tumor/control que vayas a usar.
  • Etiquetas: el preentrenamiento no las da; necesitas un set etiquetado para el fine-tuning. Verifica las de FOMO260K antes de comprometerte (Opción B); si no, usa la Opción A.
  • Licencia: FOMO260K es CC-BY-NC-SA-4.0 (no comercial) → válido para un TFG; cita la fuente.
  • Curva de aprendizaje: foundation models 3D + fine-tuning es más avanzado; apóyate en el README de AMAES y en la baseline de FOMO25.
